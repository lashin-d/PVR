# Virtual Reality Programming. 2025

- Програмна реалізація кожного завдання з номером _**#NN-Y**_ формується у окремому каталозі _**TaskNN-Y**_ або у загальному каталозі _**Tasks**_.<br>
- Файли _**TaskNN-Y\Readme.md**_ до кожного виконаного завдання повинин містити щонайменше екранні форми з прикладом функціонування програми (якщо у завданні не вказано інше).<br><hl>
- Будь ласка, дотримуйтесь коректного іменування файлів та каталогів (див. приклади нижче).<br>
- [Шаблон звіту](https://drive.google.com/drive/folders/1ikYoyCkf_dxbLNP33w4JwLqAA7KuPFaq?usp=sharing)

#### Repository directory structure example
```
 . Root of repository 
 ├── .gitignore               │ Налаштування синхронізації віддаленого репозиторію Unity (https://github.com/github/gitignore/tree/main). 
 ├── README.md                │ Файл-пам'ятка, яку Ви зараз читаєте. Зміни не потрібні. 
 ├── Tasks                    │ Загальний каталог з реалізацією всіх практичних робіт. 
 │   │
 │   ├── Task01-1             │ Програмна реалізація завдання #01-1
 │   .   ├── Readme.md        │ Екранні форми із прикладами/тестами виконання програми з короткими поясненнями за завданням #01.1.
 │   .   └── *.*              │ Файли проекту за завданням #01-1.
 │   .   
 │   └── TaskNN-Y             │ Програмна реалізація завдання #NN-Y.
 │       ├── Readme.md        │ Екранні форми із прикладами/тестами виконання програми з короткими поясненнями за завданням #NN-Y.
 │       └── *.*              │ Файли проекту за завданням #NN-Y.
 │   
 └── #NN-Y GGG ПрізвищеІБ.docx│ Єдиний загальний звіт щодо виконання практичних робіт.
                              │ де NN   – дві цифри, тема останньої повністю зробленої та оформленої роботи;
                              │    Y    – одна цифра, номер роботи за темою NN;
                              │    GGG  – номер групи, три цифри без літер.
                              │ Наприклад: "#02-1 641 П'яточкинПП.docx" - звіт з практичними роботами Петріка П'яточкина з 641п групи.
```
